# Release Guide: v0.1.0

This document defines the source-release flow for `v0.1.0` on GitHub and keeps npm publishing as a separate future step.

## 1. Verify local state before tagging

From repository root:

1. Ensure the working tree is clean:
   - `git status --short`
2. Run release validation commands:
   - `npm run lint`
   - `npm test`
   - `npm run test:coverage`
   - `npm run build`
3. Confirm changelog has `v0.1.0` section:
   - `CHANGELOG.md`

## 2. Create and push tag `v0.1.0`

1. Create annotated tag:
   - `git tag -a v0.1.0 -m "v0.1.0"`
2. Push the tag:
   - `git push origin v0.1.0`

## 3. Create GitHub Release from `CHANGELOG.md`

1. Open GitHub Releases page for the repository.
2. Create a new release for tag `v0.1.0`.
3. Set release title to `v0.1.0`.
4. Copy the `## v0.1.0 - 2026-03-04` section from `CHANGELOG.md` into release notes.
5. Publish the GitHub Release.

## 4. Future npm publish step (not part of this release)

When npm release is enabled:

1. Verify package contents:
   - `npm pack --dry-run`
2. Authenticate to npm:
   - `npm whoami`
3. Publish package:
   - `npm publish --access public`

Do not run `npm publish` as part of the current GitHub-only `v0.1.0` release.

## 5. Automated npm tarball smoke evidence

Automated release smoke for npm artifact is covered by `tests/release-npm-artifact.test.ts`.
It validates the full pre-publish path:

1. Build release artifact (`npm run build`).
2. Create tarball (`npm pack --json --pack-destination <tmp>`).
3. Verify tarball content includes release files:
   - `package.json`
   - `README.md`
   - `LICENSE`
   - `CHANGELOG.md`
   - `dist/cli.js`
   - `dist/prompts/review-agents/*`
4. Verify source-only paths are not shipped (`src/**`, `tests/**`).
5. Install tarball into a clean temporary project (`npm install --no-save <tarball>`).
6. Run CLI from installed artifact (`node_modules/.bin/thred --help`) with exit code `0`.

Result status:
- Latest local verification: `2026-03-04` -> `PASS` (`npm test`, case `release npm artifact > builds, packs, validates tarball contents, installs it, and runs CLI help`).
- CI expectation: same test must stay green on release branch before tagging.

## 6. Release readiness criteria for `v0.1.0`

Release is ready only when all checks below are true:

1. Mandatory quality gates pass:
   - `npm run lint`
   - `npm test`
   - `npm run test:coverage`
   - `npm run build`
2. npm tarball smoke test passes (`tests/release-npm-artifact.test.ts`).
3. `package.json` release metadata checks pass for:
   - `bin`
   - `files`
   - `repository`
   - `bugs`
   - `homepage`
   - `license`
   - `engines`
4. No `critical` or `high` findings remain in release review outputs.
